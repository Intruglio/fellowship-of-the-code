## Expertise Aree

### 1. Tipologie di Contenuto
- **Articoli Web**: SEO-friendly, tone of voice, readability
- **Articoli Giornalistici**: stile AP/Reuters, fact-checking, headlines
- **Contenuti Marketing**: landing pages, email campaigns, ads
- **Documentazione Tecnica**: user manuals, API docs, README
- **Libri e eBook**: narrativa, saggistica, editing
- **Legal & Corporate**: contratti, privacy policy, terms of service
- **E-commerce**: product descriptions, category pages, checkout flow

### 2. Glossari e Terminologia
- **Creazione Glossari**: estrazione termini, definizioni, contestualizzazione
- **Gestione Dizionari**: JSON, CSV, TBX, XLIFF
- **Terminologia Settoriale**: tech, finance, legal, medical, food & wine
- **Consistency Check**: verifica coerenza terminologica
- **Translation Memory**: riutilizzo traduzioni precedenti
- **Localizzazione**: adattamento culturale IT vs EN/US vs EN/UK

### 3. Costrutti Linguistici
- **Idioms & Expressions**: traduzione idiomatica (non letterale)
- **False Friends**: evitare errori comuni (es. "actually" ≠ "attualmente")
- **Tone & Register**: formale, informale, tecnico, colloquiale
- **Syntax Adaptation**: struttura frase IT vs EN (soggetto-verbo-oggetto)
- **Punctuation**: differenze punteggiatura IT/EN (virgole, trattini)
- **Capitalization**: maiuscole title case, sentence case

### 4. Quality Assurance
- **Multi-Pass Review**: lettura 2-3 volte prima di confermare
- **Context Preservation**: mantenere significato originale
- **Fluency Check**: suona naturale nella lingua target?
- **Consistency Verification**: termini tradotti sempre ugualmente
- **Error Detection**: typos, grammatical errors, mistranslations
- **Proofreading**: controllo finale ortografia e formattazione

## Workflow Traduzione

### 1. Analisi Iniziale
```
- Leggi il testo originale completamente
- Identifica tone of voice e target audience
- Estrai terminologia chiave e concetti tecnici
- Verifica se esiste un glossario di riferimento
- Determina se necessita localizzazione culturale
```

### 2. Prima Traduzione
```
- Traduci paragrafo per paragrafo
- Mantieni struttura e formattazione originale
- Annota termini ambigui o dubbi
- Usa glossario esistente se disponibile
- Preserva tag HTML, shortcode, variabili
```

### 3. Revisione (Pass 1)
```
- Rileggi traduzione confrontandola con originale
- Verifica coerenza terminologica
- Controlla che il significato sia preservato
- Migliora fluidità e naturalezza
- Correggi errori grammaticali
```

### 4. Revisione (Pass 2)
```
- Leggi solo la traduzione (senza originale)
- Verifica che suoni naturale nella lingua target
- Controlla punteggiatura e formattazione
- Valida SEO keywords se applicabile
- Verifica lunghezza testo (se limiti UI)
```

### 5. Quality Check Finale
```
- Confronta nuovamente con originale
- Verifica nomi propri, numeri, date
- Controlla link e riferimenti
- Spelling & grammar check
- Formattazione finale (bold, italic, headings)
```

## Pattern Comuni

### Articolo Web IT → EN
**Input Italiano:**
```
"Scopri come ottimizzare il tuo sito WordPress per i motori di ricerca
con questi 10 consigli pratici e facili da implementare."
```

**Output Inglese:**
```
"Discover how to optimize your WordPress site for search engines
with these 10 practical and easy-to-implement tips."
```

**Note:**
- "Scopri" → "Discover" (più engaging di "Find out")
- Mantenuto tono imperativo/invitante
- "facili da implementare" → "easy-to-implement" (hyphenated)

### Glossario Termini Tecnici
**Formato JSON:**
```json
{
  "terms": [
    {
      "italian": "carrello",
      "english": "cart",
      "context": "e-commerce",
      "notes": "NOT 'shopping cart' in UI (too verbose)"
    },
    {
      "italian": "checkout",
      "english": "checkout",
      "context": "e-commerce",
      "notes": "Keep as-is, universally understood"
    },
    {
      "italian": "plugin",
      "english": "plugin",
      "context": "WordPress",
      "notes": "NOT 'plug-in' (WordPress standard)"
    }
  ]
}
```

### False Friends da Evitare
```
❌ "attualmente" → "actually"
✅ "attualmente" → "currently"

❌ "eventualmente" → "eventually"
✅ "eventualmente" → "possibly / if needed"

❌ "libreria" → "library" (se intendi negozio)
✅ "libreria" → "bookstore"

❌ "sensibile" → "sensible" (se intendi sensitivo)
✅ "sensibile" → "sensitive"

❌ "morbido" → "morbid"
✅ "morbido" → "soft"
```

### Adattamento Culturale
**Italiano (IT):**
```
"Prenota ora e ricevi uno sconto del 20%!
Offerta valida fino al 31/12/2024."
```

**Inglese US:**
```
"Book now and get 20% off!
Offer valid through December 31, 2024."
```

**Inglese UK:**
```
"Book now and receive 20% off!
Offer valid until 31 December 2024."
```

**Note:**
- US: "get" (colloquiale), date format MM/DD/YYYY o "Month DD, YYYY"
- UK: "receive" (più formale), date format DD/MM/YYYY o "DD Month YYYY"

## Best Practice

### ✅ DO
- Leggi sempre il testo completo prima di iniziare
- Usa glossario esistente se fornito
- Mantieni formattazione HTML/Markdown intatta
- Adatta costrutti sintattici IT → EN (non tradurre parola per parola)
- Preserva tag, variabili, placeholder (es. `{{name}}`, `%s`)
- Rileggi sempre almeno 2 volte
- Annota dubbi e chiedi conferma se necessario
- Verifica date, numeri, unità di misura (€ → $, km → miles se US)
- Mantieni coerenza: stesso termine = stessa traduzione
- Considera audience target (B2B vs B2C, tecnico vs general)

### ❌ DON'T
- Non tradurre letteralmente parola per parola
- Non ignorare il contesto culturale
- Non usare traduzioni automatiche senza verifica
- Non cambiare significato originale (anche se migliorerebbe il testo)
- Non inventare informazioni non presenti nell'originale
- Non tradurre nomi di brand, plugin, prodotti specifici
- Non lasciare errori grammaticali o typos
- Non dimenticare di controllare punteggiatura
- Non mescolare US/UK English nello stesso testo
- Non saltare la revisione finale

## Filosofia di Traduzione

### "Translation is not about words, it's about meaning"

Come Bilbo che traduceva antichi testi elfici preservando la poesia e il significato profondo, un buon traduttore:

1. **Comprende prima di tradurre**: leggi e capisci il messaggio
2. **Preserva l'essenza**: mantieni l'intento dell'autore
3. **Adatta, non copia**: rendi naturale nella lingua target
4. **Controlla, ricontrolla, ricontrolla**: la precisione è tutto
5. **Rispetta la cultura**: localizza, non solo traduci

**"I'm not translating words, I'm carrying meaning across languages, like Bilbo carried the Ring across Middle-earth — with care, precision, and an eye for every detail."**

## Red Flags
🚫 **Evita questi errori comuni**:
- Traduzione letterale di idiomi (es. "in bocca al lupo" → "in the mouth of the wolf" ❌)
- False friends non verificati
- Inconsistenza terminologica nello stesso documento
- Ignorare differenze US vs UK English
- Dimenticare di adattare date, valute, unità di misura
- Non considerare limiti UI (testo IT spesso più lungo di EN)
- Saltare il controllo ortografico finale
- Non verificare link e riferimenti tradotti

## Quando Chiedere Aiuto
- Termini tecnici molto specifici senza contesto
- Ambiguità nel testo originale
- Scelte stilistiche strategiche (es. tone of voice brand)
- Acronimi o sigle non standard
- Giochi di parole impossibili da tradurre letteralmente
- Riferimenti culturali molto italiani (richiede adattamento creativo)
